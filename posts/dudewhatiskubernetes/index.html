Dude what is Kubernetes?

A brief explanation of what Kubernetes is and how to get started
March 20, 2022
Reading Time: 4 mins

This week, I began tinkering with Kubernetes for a data science related project. Prior to this, I would always hear of this seemingly alien technology and its ability to cause excitement amongst the most elitist developers. Kubernetes was supposedly everywhere and knowing how to use it was a resume worthy addition.

There was no way to stop it’s eventual rule over the world.

Contrastingly, I’ve also seen fellow software enthusiasts with their face in their hands, turning more red than the ripest tomato you’ve ever seen due to sheer frustration with the tool (then again, this is in the software developer job description).

With fancy new terms that I didn’t understand including clusters, nodes and pods, I decided never to use Kubernetes (or K8s as the cool kids call it) in my personal projects simply because it felt unecessary (and hopefully by the end of this article you’ll understand why). To some extent, I was even fearful of the tool and the hours I might need to spend debugging a complex problem that could be caused by one of a million things.

Despite this, I started reading into what it is and why it is used. Whilst I’m still fairly new to using K8s, it became clear to me that the concept behind this innovation was to serve a clear purpose, one of which I hope you’ll better understand after this read.

For those who don’t live in a container…#

Before I go into Kubernetes, I feel like it’s worth first explaining what containerised applications are.

Whenever we want to deploy a piece of software, we are tasked with several issues including package management and environment setup.

Containerization is a software development concept which involves taking an application and instead isolating its execution into a virtual environment known as a container. Using containers allows us to run our software in a more controlled environment, reducing potential deployment issues along the way. This approach to building software can also improve portability as applications within containers can be run as images, usable in any environment with the correct setup for using containers.

Applications such as Docker allow for container management and have particularly useful features including the ability to remove containers after they have finished being run and interactive modes to enter the container environment.

I might do another article in future to rant about my passion for containers but for now, I’ll leave it at what has been explained so far :)

So where does K8s fit in?#

Whilst Docker allows you to run and manage your containers, it is limited to being designed for use on a single machine (eg: My MacBook Air). This is brilliant for hobbyists like me but…

It is highly unusable for parties which wish to use large and numerous amounts of containers to perform tasks

The power of Kubernetes lies in its ability to orchestrate deployment of containerised applications within a distributed computing system, making it possible to run thousands of hefty apps all at once over several machines.

As you can probably imagine, this is a pretty impressive feat and a brilliant feature for many large corporates to have.

Let’s look at the basic structure#

K8s is comprised of some critical components:

  • Cluster(s): These are groups of several machines working together to run Kubernetes

  • Control Plane: This is a program which is responsible for managing the “health” of your cluster and providing features such as fault tolerance

  • Worker nodes: These are individual machines (either virtual or physical) which are capable of running your actual containerized application

  • Pods: These essentially run worker nodes in groups, helping you to have further control over the arrangement of your applications

To get started with setting up a K8s cluster locally for experimenting, check out the docs for a tool called minikube

Another critical tool we use to interact with our K8s cluster is kubectl. Alongside the programmer, these tools and structure to how K8s operates helps us to better control our running of containerised applications on large systems.
